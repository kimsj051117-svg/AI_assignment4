# LabLog

**Completed for Assignment 4**

LabLog는 학생 연구자가 연구 프로젝트, 실험 기록, 문헌, 실험 조건 템플릿, 실험 비교, 결과 아카이브를 한 곳에서 관리할 수 있도록 만든 한국어 기반 정적 웹 연구노트 서비스입니다.

이 프로젝트는 **Introduction to AI Programming - Assignment 4** 제출을 위해 구현되었습니다.

---

## 1. 프로젝트 개요

LabLog는 반복 실험을 수행하는 학생 연구자를 위한 연구 워크스페이스입니다. 일반적인 메모 앱은 실험 조건, 변경 변수, 결과 파일, 참고 문헌을 연결해서 관리하기 어렵습니다. LabLog는 연구 프로젝트별로 노트와 실험 기록을 정리하고, 조건 템플릿을 활용해 실험 조건의 변화를 추적할 수 있도록 설계되었습니다.

주요 대상 사용자는 다음과 같습니다.

- 실험 조건을 반복적으로 바꾸며 결과를 비교해야 하는 학생 연구자
- 프로젝트별 연구 노트와 문헌을 함께 관리하고 싶은 사용자
- 서버나 로그인 없이 브라우저에서 간단히 연구 데이터를 저장하고 싶은 사용자

---

## 2. 주요 기능

### 연구 프로젝트 관리

- 연구 프로젝트를 추가, 수정, 삭제할 수 있습니다.
- 프로젝트명, 연구 분야, 목표, 시작일, 상태, 키워드, 메모를 관리합니다.
- 프로젝트 이름 변경 시 연결된 연구 노트, 실험 기록, 문헌, 결과 기록도 함께 갱신됩니다.

### 연구 노트 / 실험 기록 작성

- 새 연구 노트와 실험 기록을 작성할 수 있습니다.
- 제목, 프로젝트, 날짜, 목적, 상태, 결과 요약, 메모, 관련 문헌, 키워드를 입력할 수 있습니다.
- 결과 파일을 첨부할 수 있으며, 저장된 실험 기록은 결과 아카이브와 연결됩니다.
- 기존 기록을 수정, 삭제, 복제할 수 있습니다.

### 상황별 실험 템플릿

- 일반 실험 노트, 합성 실험, 측정/분석 실험, 소재 제작, 조건 최적화, 문헌 기반 실험 계획 등 상황별 템플릿을 제공합니다.
- 템플릿을 선택하면 실험 상황에 맞는 입력 항목을 빠르게 작성할 수 있습니다.

### 실험 조건 템플릿

- 반복 실험에 사용할 기본 조건 세트를 저장할 수 있습니다.
- 변수명과 기본값을 등록해 이후 실험 기록에서 재사용할 수 있습니다.
- 템플릿을 선택하면 기본 조건이 자동 입력됩니다.

### 변경 변수 추적

- 기본 조건 템플릿과 실제 실험 조건을 비교해 변경된 변수만 강조 표시합니다.
- 어떤 조건이 이전 실험 또는 기본 조건과 달라졌는지 빠르게 확인할 수 있습니다.

### 실험 비교

- 여러 실험 기록을 선택해 조건과 결과를 나란히 비교할 수 있습니다.
- 변경된 변수는 표에서 강조되어 실험 차이를 쉽게 파악할 수 있습니다.
- 이전 실험을 복제해 새로운 실험 기록을 만들 수 있습니다.

### 문헌 아카이브

- 논문 제목, 저자, 연도, 출처, DOI/URL, 관련 프로젝트, 키워드, 요약, 개인 노트를 저장할 수 있습니다.
- 문헌 기록을 추가, 수정, 삭제할 수 있습니다.
- DOI 입력 시 문헌 정보를 자동으로 불러오는 기능을 제공합니다.

### 결과 아카이브

- 연구 노트와 연결된 실험 결과, 독립적으로 추가한 결과 기록, 첨부 파일을 관리합니다.
- 결과 값이 있는 기록은 간단한 추이 차트로 확인할 수 있습니다.
- 프로젝트, 제목, 주요 변수 기준으로 결과를 그룹화해 볼 수 있습니다.

### 검색, 필터, 정렬

- 연구 노트는 제목, 메모, 프로젝트, 목적, 키워드, 문헌, 상태, 템플릿명, 결과 내용을 기준으로 검색할 수 있습니다.
- 상태 필터와 날짜 정렬을 지원합니다.
- 결과 아카이브와 문헌 아카이브에서도 검색 기능을 제공합니다.

### localStorage 저장

- 모든 데이터는 브라우저 `localStorage`에 저장됩니다.
- 별도의 로그인, 서버, 데이터베이스 없이 사용할 수 있습니다.
- JSON 백업 파일로 데이터를 내보내고 다시 가져올 수 있습니다.

## 3. 기술 스택

- **HTML**: 웹 페이지 구조와 모달, 입력 폼 구성
- **CSS**: 반응형 레이아웃, 카드 UI, 대시보드 스타일, 모달 디자인
- **JavaScript**: 데이터 관리, 화면 렌더링, 검색/필터/정렬, 실험 비교 로직
- **Browser localStorage**: 프로젝트, 노트, 문헌, 템플릿, 결과 데이터 저장
- **Vercel**: 정적 웹사이트 배포

## 4. 파일 구조

```text
lablog/
├── index.html
├── style.css
├── script.js
└── README.md
```

### `index.html`

웹사이트의 기본 HTML 구조, 사이드바, 상단 영역, 탭, 입력 폼, 도움말 모달, 데이터 관리 모달을 포함합니다.

전체 UI 스타일을 정의합니다. 배경, 사이드바, 카드, 버튼, 탭, 모달, 반응형 레이아웃 등 시각적 디자인을 담당합니다.

### `script.js`

프로젝트 관리, 연구 노트 작성, 실험 조건 템플릿, 변경 변수 추적, 실험 비교, 문헌 아카이브, 결과 아카이브, JSON 백업, localStorage 저장 로직을 담당합니다.

### `README.md`

과제 제출용 프로젝트 설명 문서입니다.

## 7. 사용 흐름

1. `연구 프로젝트`에서 연구 주제 또는 실험 프로젝트를 생성합니다.

7. `문헌 아카이브`에서 관련 논문과 참고 자료를 정리합니다.
8. `결과 아카이브`에서 실험 결과와 첨부 파일을 관리합니다.
9. `데이터 관리`에서 JSON 파일로 데이터를 백업하거나 복원합니다.

## 8. 데이터 저장 방식

LabLog는 서버나 데이터베이스를 사용하지 않고 브라우저 `localStorage`에 데이터를 저장합니다.

저장되는 주요 데이터는 다음과 같습니다.

- 연구 프로젝트
- 연구 노트 및 실험 기록
- 문헌 기록
- 상황별 실험 템플릿 입력값
- 실험 조건 템플릿
- 결과 기록
- 첨부 파일 데이터

브라우저 저장소를 삭제하거나 다른 기기에서 접속하면 기존 데이터가 보이지 않을 수 있습니다. 중요한 데이터는 `데이터 관리` 기능을 통해 JSON 파일로 백업하는 것을 권장합니다.

### AI 결과에서 직접 수정한 부분

- 과제 요구사항에 맞게 기능 범위를 조정했습니다.
- 자동 생성된 설명 중 실제 구현과 맞지 않는 표현을 삭제했습니다.
- UI 디자인은 제안된 스타일을 바탕으로 직접 선택자를 확인하며 기존 기능이 깨지지 않도록 적용했습니다.
- localStorage 기반 정적 웹사이트라는 제한을 명확히 반영했습니다.

### 발견한 문제와 해결 방법

- **문제**: 템플릿을 선택하지 않은 연구 노트에서 결과 요약 또는 첨부 파일이 결과 아카이브에 제대로 연결되지 않을 수 있었습니다.  
  **해결**: 결과 요약, 첨부 파일, 조건 행이 있으면 실험 기록을 생성하도록 저장 조건을 수정했습니다.

- **문제**: 검색 범위가 제목과 메모 중심이라 프로젝트명, 키워드, 문헌명, 결과 내용 검색이 제한적이었습니다.  
  **해결**: 연구 노트 검색 대상을 프로젝트, 목적, 키워드, 문헌, 상태, 템플릿명, 결과까지 확장했습니다.

- **문제**: 일부 카드 hover 애니메이션이 입력 폼 사용 중 산만하게 느껴졌습니다.  
  **해결**: 입력 폼과 필터 카드의 움직임을 제거하고, 일반 노트 카드에만 가벼운 hover 효과를 유지했습니다.
