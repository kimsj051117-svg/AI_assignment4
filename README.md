# LabLog

**LabLog**는 학생 연구자를 위한 한국어 기반 정적 웹 연구노트 서비스입니다. 연구 프로젝트, 연구 노트, 문헌 아카이브, 실험 조건 템플릿, 템플릿 라이브러리, 실험 비교, 결과 아카이브를 한 화면에서 관리할 수 있습니다.

이 프로젝트는 **Introduction to AI Programming - Assignment 4** 과제 제출을 위해 구현되었습니다.

## 주요 기능

### 1. 연구 프로젝트

- 프로젝트명, 연구 분야, 연구 목표, 시작일, 상태, 키워드, 메모 저장
- 프로젝트 카드 목록 표시
- 프로젝트 추가, 수정, 삭제
- 프로젝트명을 수정하면 연결된 연구노트, 실험 기록, 문헌, 결과 기록의 프로젝트명도 함께 갱신
- 프로젝트 삭제 시 연결된 데이터는 `미분류`로 이동

### 2. 연구 노트 / 새 실험 기록

- 새 실험 기록 작성
- 저장된 프로젝트 선택
- 상황별 템플릿 선택
- 기본 실험 조건 템플릿 선택
- 결과 요약 및 결과 파일 첨부
- 연구노트 카드 표시
- 연구노트 수정, 삭제, 복제
- 검색, 상태 필터, 날짜 정렬
- 작성 중인 모달을 닫을 때 저장되지 않은 내용이 있으면 확인 안내 표시

### 3. 템플릿 라이브러리

- 실험 상황에 맞는 템플릿을 미리 확인하고 선택할 수 있습니다.
- 일반 실험 노트, 합성 실험, 측정/분석 실험, 소자 제작, 조건 최적화, 문헌 기반 실험 계획 템플릿을 제공합니다.
- 템플릿을 선택하면 새 실험 기록 작성 화면에 상황별 입력 필드가 표시됩니다.

### 4. 실험 조건 템플릿

- 사용자가 기본 실험 조건 변수를 직접 추가하거나 삭제할 수 있습니다.
- 변수명과 기본값을 저장할 수 있습니다.
- 새 실험 기록에서 기본 조건 템플릿을 선택하면 조건이 자동 입력됩니다.
- 기본값과 달라진 변수는 변경 항목으로 강조됩니다.

### 5. 변경 변수 추적

LabLog는 기본 조건과 실제 실험 조건을 비교해 변경된 변수만 강조합니다.

예시:

```text
Reaction Temperature: 80 C -> 90 C
Reaction Time: 2 h -> 3 h
```

### 6. 실험 비교

- 저장된 실험 기록을 표 형태로 비교
- 비교할 실험을 2개 이상 선택해 변수별 side-by-side 비교 가능
- 값이 다른 변수 행은 강조 표시
- 결과 요약 행 포함
- 이전 실험 복제 기능 제공

### 7. 그룹 보기

연구노트 목록에서 보기 방식을 선택할 수 있습니다.

- 전체 기록
- 프로젝트별 보기
- 실험 제목별 보기
- 주요 변수별 보기

그룹 보기에서도 검색, 상태 필터, 날짜 정렬을 적용할 수 있습니다.

### 8. 문헌 아카이브

- 논문 제목, 저자, 연도, 출처, DOI 또는 URL, 관련 프로젝트, 키워드, 요약, 개인 노트 저장
- 문헌 기록 추가, 수정, 삭제
- 제목, 저자, 키워드, 프로젝트 기준 검색
- DOI를 입력하고 `DOI 정보 불러오기`를 누르면 Crossref API를 통해 기본 문헌 정보를 자동 입력
- URL이 있으면 클릭 가능한 링크로 표시

### 9. 결과 아카이브

- 실험 결과 요약과 첨부 파일 확인
- 독립 결과 또는 연구노트에 연결된 결과를 구분해 표시
- 결과 기록 추가, 수정, 삭제
- 결과값 추이 차트 표시
- 프로젝트명 또는 주요 변수 검색
- 보기 방식에 따라 프로젝트별, 제목별, 주요 변수별 그룹 표시

### 10. 도움말 보기

- 상단의 `도움말 보기` 버튼을 통해 LabLog 사용 순서를 확인할 수 있습니다.
- 처음 사용하는 사용자는 연구 프로젝트 생성, 템플릿 선택, 새 실험 기록 작성, 실험 비교, 데이터 백업 순서로 서비스를 사용할 수 있습니다.
- 별도의 로그인 없이 localStorage 기반으로 데이터를 관리하는 방식도 안내합니다.

### 11. 데이터 관리

- 상단의 `데이터 관리` 버튼을 통해 데이터 백업 및 복원 모달을 열 수 있습니다.
- JSON 백업 파일 다운로드
- JSON 백업 파일 가져오기
- 가져오기 전 기존 데이터 덮어쓰기 확인
- 잘못된 JSON 파일 또는 LabLog 백업 형식이 아닌 파일에 대한 오류 안내

### 12. 대시보드 요약

상단 요약 카드에서 다음 정보를 확인할 수 있습니다.

- 전체 노트 수
- 연구 프로젝트 수
- 실험 기록 / 결과 기록 수
- 문헌 / 템플릿 수

## 사용 흐름

1. `연구 프로젝트`에서 실험 또는 연구 주제를 생성합니다.
2. `템플릿 라이브러리`에서 실험 상황에 맞는 템플릿을 확인합니다.
3. `연구 노트`에서 새 실험 기록을 작성합니다.
4. 필요한 경우 `실험 조건 템플릿`을 선택해 기본 조건을 자동 입력합니다.
5. 바뀐 실험 조건만 수정하면 변경된 변수가 자동으로 강조됩니다.
6. `실험 비교`에서 여러 실험을 나란히 비교합니다.
7. `결과 아카이브`에서 결과값과 첨부 파일을 관리합니다.
8. `데이터 관리`에서 JSON 백업 파일을 다운로드하거나 가져옵니다.

## 데이터 저장

모든 데이터는 브라우저 `localStorage`에 저장됩니다.

주요 저장 데이터:

- 연구 프로젝트
- 연구노트
- 문헌 기록
- 기본 실험 조건 템플릿
- 실험 기록
- 결과 기록

브라우저 저장소를 삭제하면 저장된 데이터도 삭제됩니다.

---

## 데이터 내보내기/가져오기

LabLog는 localStorage 기반 서비스이므로 모든 데이터는 현재 브라우저에 저장됩니다.

상단의 `데이터 관리` 버튼을 누르면 데이터 관리 모달이 열립니다.

사용자는 이 모달에서 다음 기능을 사용할 수 있습니다.

- `백업 파일 다운로드`: 연구 프로젝트, 연구노트, 문헌, 템플릿, 실험 기록, 결과 아카이브를 JSON 파일로 내보냅니다.
- `백업 파일 가져오기`: 이전에 내보낸 LabLog JSON 백업 파일을 불러와 데이터를 복원합니다.

가져오기를 하면 기존 브라우저 데이터가 덮어써질 수 있으므로, 중요한 기록이 있다면 먼저 백업 파일을 다운로드하는 것을 권장합니다.

## 파일 구조

```text
lablog/
├── index.html
├── style.css
├── script.js
└── README.md
```

### `index.html`

웹사이트의 기본 구조, 사이드바, 상단 탭, 새 실험 기록 모달, 도움말 모달, 데이터 관리 모달을 포함합니다.

### `style.css`

색상, 배경, 버튼, 카드, 표, 모달, 차트, 반응형 레이아웃 등 UI 스타일을 정의합니다.

### `script.js`

다음 기능을 처리합니다.

- 프로젝트 추가/수정/삭제
- 문헌 추가/수정/삭제 및 DOI 조회
- 상황별 템플릿 적용
- 기본 조건 템플릿 관리
- 연구노트 추가/수정/삭제/복제
- 변경 변수 추적
- 그룹 보기
- 실험 비교
- 결과 아카이브
- 결과값 추이 차트
- 도움말 모달 열기/닫기
- 데이터 관리 모달 열기/닫기
- JSON 데이터 내보내기/가져오기
- localStorage 저장 및 불러오기

## 로컬 실행 방법

### 방법 1: 브라우저에서 직접 열기

1. 프로젝트 폴더를 엽니다.
2. `index.html`을 더블클릭합니다.
3. 웹사이트가 브라우저에서 실행됩니다.

### 방법 2: VS Code Live Server 사용

1. VS Code에서 프로젝트 폴더를 엽니다.
2. Live Server 확장을 설치합니다.
3. `index.html`을 우클릭합니다.
4. **Open with Live Server**를 선택합니다.

---

## 테스트 방법

1. Live Demo 링크에 접속합니다.
2. 상단의 `도움말 보기` 버튼을 눌러 사용 순서를 확인합니다.
3. `연구 프로젝트`에서 새 프로젝트를 추가합니다.
4. `템플릿 라이브러리`에서 상황별 템플릿을 선택합니다.
5. `연구 노트`에서 새 실험 기록을 작성합니다.
6. 기본 조건 템플릿을 선택하고 일부 변수만 수정해 변경 표시를 확인합니다.
7. `실험 비교`에서 2개 이상의 실험을 선택해 비교합니다.
8. `결과 아카이브`에서 결과 기록과 결과값 추이를 확인합니다.
9. 상단의 `데이터 관리` 버튼을 눌러 JSON 백업 파일을 다운로드합니다.
10. 백업 파일 가져오기 기능으로 JSON 파일 복원이 가능한지 확인합니다.
